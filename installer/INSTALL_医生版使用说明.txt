【系统要求 / Requirements】
────────────────────────────────────────────────────────────
· Windows 10 / 11（64位）
· 摄像头（内置或外置均可，USB 摄像头推荐 720p 以上）
· 网络连接（仅首次安装需要）
· 建议内存：8 GB 以上

────────────────────────────────────────────────────────────
【首次安装 / First-time Installation】
────────────────────────────────────────────────────────────
1. 双击运行  →  setup_windows.bat
2. 等待自动下载 Python 和依赖包（约 3–5 分钟，取决于网速）
3. 安装完成后，桌面会自动出现两个快捷方式：
      ▶ "EyeQ 仪表盘"         （推荐，浏览器可视化界面）
      ▶ "EyeQ 命令行监测"     （纯文字输出，数据更精简）

注意：安装过程中可能弹出"Windows 安全警告"，
      请点击"仍要运行"或"允许访问"。

────────────────────────────────────────────────────────────
【日常使用 / Daily Use】
────────────────────────────────────────────────────────────
▸ 启动仪表盘：
  双击桌面"EyeQ 仪表盘" → 浏览器自动打开 http://localhost:8501
  页面显示实时 EAR 曲线、眨眼频率、泪膜风险评分等指标。

▸ 启动命令行模式（更轻量）：
  双击桌面"EyeQ 命令行监测" → 弹出黑色命令行窗口，显示实时表格

▸ 停止程序：
  在命令行窗口或浏览器标签页中按 Ctrl+C，或直接关闭窗口。

▸ 会话数据保存位置：
  data\eyeq_sessions.db（SQLite 数据库，可用 DB Browser for SQLite 查看）

【屏幕各项指标说明 / Metric Reference】
────────────────────────────────────────────────────────────
指标名称                 参考范围              临床意义
──────────────────────── ──────────────────── ────────────────────────────
眨眼频率 Blink Rate      15–25 次/分钟         < 10 提示低频（屏幕疲劳）
                                               > 30 提示代偿性高频（干眼刺激）

不完全眨眼比 IBR         < 30%                 ≥ 40% 与睑板腺功能障碍相关
估算 NIBUT               3–7 秒                < 3s 泪膜不稳定；> 8s 或有抑制
IBI 变异系数 CV          < 0.4                 > 0.6 提示眨眼节律紊乱

综合风险评分             0–30   低风险（绿）   眼部状态良好，建议继续观察
                         31–60  中风险（橙）   建议行 OSDI 问卷 + 裂隙灯检查
                         61–100 高风险（红）   建议完整干眼评估（含 Schirmer）

【常见问题 / FAQ】
────────────────────────────────────────────────────────────
Q：摄像头无画面 / 显示"无法读取摄像头"？
A：请打开 Windows 设置 → 隐私和安全 → 摄像头，
   确认"允许应用访问摄像头"已开启，并允许命令提示符访问。

Q：仪表盘页面空白或加载中？
A：等待约 10 秒让模型加载完毕，或刷新浏览器页面。

Q：杀毒软件拦截了程序？
A：将 EyeQ 文件夹添加到杀毒软件的白名单/排除列表中。

Q：如何更换摄像头？
A：用文字编辑器打开 config.yaml，修改 camera.source 的数字
   （0=第一个摄像头，1=第二个，依此类推）。
